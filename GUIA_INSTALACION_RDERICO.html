ERP R DE RICO

Guía Maestra de Instalación

AVISO DE ACTUALIZACIÓN: El equipo tiene una versión previa. Esta instalación es una sustitución limpia. No se conservan datos anteriores. Se elimina todo rastro de contenedores y volúmenes antes de iniciar.

Contexto del Proyecto

Este sistema es un ERP Punto de Venta diseñado para operación en red local con múltiples terminales (6 CPUs), optimizado para hardware de bajas especificaciones y estaciones de cobro centralizado.

PARTE 1: Instalación en el Servidor (CPU 1)

Coexistencia con SAS: El servidor convive con un sistema SAS. El ERP corre aislado en Docker. No toque el SAS. Verifique que los puertos 3000 y 3001 estén libres con netstat -ano | findstr ":3000 :3001".

Paso 1: Parada y Limpieza Profunda

Borre todo el sistema anterior antes de clonar el nuevo.
docker stop rderico-pos rderico-api-edge rderico-db-edge
docker rm rderico-pos rderico-api-edge rderico-db-edge
docker volume rm erp-r-de-rico_pgdata

Paso 2: Obtención del Código

git clone https://github.com/manualesrderico-hash/ERP-R-DE-RICO.git
cd ERP-R-DE-RICO

Paso 3: Despliegue

docker-compose up -d --build

Paso 4: Carga de Datos

# Verifique http://localhost:3001/health primero
python import_data.py

PARTE 2: Configuración de Clientes (CPU 2-6)

Acceda desde cualquier navegador en los otros equipos usando la dirección IP del servidor:

http://[IP_DEL_SERVIDOR]:3000
T2 - T5: Navegador directo.
CAJA: Navegador + Impresión Kiosk.

PARTE 3: Impresora Térmica (CAJA)

En el CPU 6, configure Chrome/Edge con impresión silenciosa:

"chrome.exe" --kiosk-printing

Esto permite que el ticket de 80mm salga instantáneamente al cobrar.
